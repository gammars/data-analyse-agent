# LangChain 数据分析智能体项目技术文档

生成日期：2026-06-26
项目定位：从 0 开始新建一个基于 LangChain / LangGraph 的数据分析智能体项目
项目目标：实现一个支持数据上传、数据理解、自然语言分析、SQL 查询、图表生成、分析报告导出的智能数据分析系统

------

# 1. 项目总目标

本项目旨在从零构建一个 **LangChain 数据分析智能体系统**。

用户可以上传 CSV、Excel 等结构化数据文件，然后通过自然语言与系统对话，例如：

- “这个数据集有哪些字段？”
- “帮我统计各地区销售额”
- “画一张销售额变化趋势图”
- “分析哪些因素影响利润”
- “生成一份数据分析报告”

系统需要自动完成以下工作：

1. 读取并理解用户上传的数据。
2. 自动生成数据表结构说明。
3. 根据用户问题选择合适的分析工具。
4. 使用 SQL、Python 或统计方法完成分析。
5. 生成可视化图表。
6. 输出自然语言解释。
7. 支持导出分析报告。

本项目不是对已有项目进行改造，而是从零开始搭建一个新的 LangChain Agent 项目。

## 2.2 自然语言数据问答能力

用户可以直接用自然语言提出分析需求。

例如：

```text
帮我看看这个数据集主要有哪些字段？
统计每个类别的销售额总和。
找出销售额最高的前 10 个商品。
```

智能体需要判断：

1. 用户是否在问数据结构。
2. 用户是否需要执行 SQL 查询。
3. 用户是否需要调用 Python 分析。
4. 用户是否需要生成图表。
5. 用户是否需要导出报告。
6. 用户问题是否模糊，是否需要追问。

------

## 2.3 SQL 查询能力

系统需要支持 Agent 根据用户问题自动生成 SQL，并在安全限制下执行查询。

例如用户问：

```text
各城市的平均销售额是多少？
```

Agent 可以生成类似 SQL：

```sql
SELECT city, AVG(sales) AS avg_sales
FROM data_table
GROUP BY city
ORDER BY avg_sales DESC;
```

SQL 执行结果返回给 Agent 后，Agent 再用自然语言解释结果。

SQL 查询必须有安全限制：

1. 只允许 SELECT 查询。
2. 禁止 DROP、DELETE、UPDATE、INSERT。
3. 禁止访问系统文件。
4. 限制返回行数。
5. 限制执行时间。
6. 对异常 SQL 返回友好错误信息。

------

## 2.4 Python 数据分析能力

对于 SQL 不方便完成的任务，系统应提供 Python 分析工具。

例如：

- 相关性分析

- 异常值检测
- 描述性统计
- 分组统计

Python 分析能力不应让模型直接执行任意代码，而是通过预定义工具函数完成。

第一阶段不建议开放任意 Python 代码执行，避免安全风险。

# 6. LangGraph 工作流设计

本项目采用 LangGraph 构建 Agent 状态图。

基础流程如下：

```text

  ↓
agent_node
  ↓
是否有 tool_calls？
  ├── 有：tool_node → agent_node
  └── 无：finish_node
  ↓
END
```

## 6.2 Agent Graph 设计

新增文件：

```text
app/agent/graph.py
```

核心代码：

```python
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode
from app.agent.state import AgentState


def should_continue(state: AgentState):
    messages = state.get("messages", [])
    last_message = messages[-1] if messages else None

    tool_calls = getattr(last_message, "tool_calls", None)

    if state.get("error"):
        return "finish"

    if tool_calls:
        return "tools"

    return "finish"


def build_agent_graph(agent_node, tools):
    graph = StateGraph(AgentState)

    graph.add_node("agent", agent_node)
    graph.add_node("tools", ToolNode(tools))
    graph.add_node("finish", lambda state: state)

    graph.set_entry_point("agent")

    graph.add_conditional_edges(
        "agent",
        should_continue,
        {
            "tools": "tools",
            "finish": "finish",
        },
    )

    graph.add_edge("tools", "agent")
    graph.add_edge("finish", END)

    return graph.compile()
```

# 9. 数据集服务设计

新增文件：

```text
app/services/dataset_service.py
```

职责：

1. 保存上传文件。
2. 读取 CSV / Excel。
3. 生成 dataset_id。
4. 保存数据集元信息。
5. 生成 schema 文本。
6. 提供样例数据。
7. 提供 DataFrame 访问接口。

核心设计：

```python
import uuid
import pandas as pd
from pathlib import Path


DATASET_DIR = Path("app/storage/datasets")


class DatasetService:
    def __init__(self):
        self.datasets = {}

    def save_dataset(self, file) -> str:
        dataset_id = str(uuid.uuid4())
        suffix = Path(file.filename).suffix.lower()

        save_path = DATASET_DIR / f"{dataset_id}{suffix}"
        save_path.parent.mkdir(parents=True, exist_ok=True)

        with open(save_path, "wb") as f:
            f.write(file.file.read())

        if suffix == ".csv":
            df = pd.read_csv(save_path)
        elif suffix in [".xlsx", ".xls"]:
            df = pd.read_excel(save_path)
        else:
            raise ValueError("暂不支持该文件类型")

        self.datasets[dataset_id] = {
            "path": str(save_path),
            "filename": file.filename,
            "df": df,
            "schema": self._build_schema(df),
        }

        return dataset_id

    def get_dataframe(self, dataset_id: str):
        return self.datasets[dataset_id]["df"]

    def get_schema(self, dataset_id: str) -> str:
        return self.datasets[dataset_id]["schema"]

    def _build_schema(self, df: pd.DataFrame) -> str:
        lines = []
        lines.append(f"行数：{len(df)}")
        lines.append(f"列数：{len(df.columns)}")
        lines.append("字段信息：")

        for col in df.columns:
            lines.append(
                f"- {col}: {df[col].dtype}, 缺失值 {df[col].isna().sum()} 个"
            )

        lines.append("\n样例数据：")
        lines.append(df.head(5).to_markdown(index=False))

        return "\n".join(lines)
```

第一阶段可以用内存字典保存数据集，后续再换成 SQLite 或文件元数据。

# 10. SQL 查询服务设计

新增文件：

```text
app/services/sql_service.py
```

建议使用 DuckDB 对 Pandas DataFrame 执行 SQL。

```python
import duckdb


class SQLService:
    def __init__(self, dataset_service):
        self.dataset_service = dataset_service

    def query(self, dataset_id: str, sql: str, max_rows: int = 100):
        self._validate_sql(sql)

        df = self.dataset_service.get_dataframe(dataset_id)

        conn = duckdb.connect(database=":memory:")
        conn.register("data_table", df)

        result = conn.execute(sql).fetchdf()

        if len(result) > max_rows:
            result = result.head(max_rows)

        return result

    def _validate_sql(self, sql: str):
        lowered = sql.lower()

        forbidden = [
            "drop ",
            "delete ",
            "update ",
            "insert ",
            "alter ",
            "create ",
            "copy ",
            "attach ",
            "install ",
            "load ",
        ]

        for word in forbidden:
            if word in lowered:
                raise ValueError(f"禁止执行危险 SQL：{word.strip()}")

        if not lowered.strip().startswith(("select", "with")):
            raise ValueError("只允许 SELECT / WITH 查询")
```

约定：用户上传的数据表统一注册为 `data_table`。

工具实现：

```python
import uuid
from pathlib import Path
import matplotlib.pyplot as plt


CHART_DIR = Path("app/storage/charts")


def make_generate_chart_tool(sql_service):
    def _run(dataset_id: str, sql: str, chart_type: str, x: str, y: str, title: str):
        result = sql_service.query(dataset_id, sql, max_rows=1000)

        chart_id = str(uuid.uuid4())
        save_path = CHART_DIR / f"{chart_id}.png"
        save_path.parent.mkdir(parents=True, exist_ok=True)

        plt.figure(figsize=(10, 6))

        if chart_type == "bar":
            plt.bar(result[x], result[y])
        elif chart_type == "line":
            plt.plot(result[x], result[y], marker="o")
        elif chart_type == "scatter":
            plt.scatter(result[x], result[y])
        elif chart_type == "pie":
            plt.pie(result[y], labels=result[x], autopct="%1.1f%%")
        else:
            raise ValueError("暂不支持该图表类型")

        plt.title(title)
        plt.xticks(rotation=45)
        plt.tight_layout()
        plt.savefig(save_path)
        plt.close()

        return {
            "chart_id": chart_id,
            "chart_path": str(save_path),
            "message": f"图表已生成：{title}",
        }

    return StructuredTool.from_function(
        name="generate_chart",
        description="根据 SQL 查询结果生成图表。",
        func=_run,
        args_schema=GenerateChartArgs,
    )
```

# 12. Agent Node 实现

新增文件：

```text
app/agent/graph.py
```

Agent 节点负责调用大模型。

```python
def make_agent_node(model, prompt, tools):
    model_with_tools = model.bind_tools(tools)
    chain = prompt | model_with_tools

    def agent_node(state):
        response = chain.invoke({
            "messages": state["messages"]
        })
        return {
            "messages": [response]
        }

    return agent_node
```

整体创建函数：

```python
from app.agent.models import build_chat_model
from app.agent.prompts import build_prompt_with_schema
from app.agent.tools import build_tools
from app.agent.graph import build_agent_graph


def create_data_analysis_agent(dataset_service, sql_service, dataset_id: str):
    model = build_chat_model()
    tools = build_tools(dataset_service, sql_service)
    schema_text = dataset_service.get_schema(dataset_id)
    prompt = build_prompt_with_schema(schema_text)

    agent_node = make_agent_node(model, prompt, tools)
    graph = build_agent_graph(agent_node, tools)

    return graph
```

# 19. 安全设计

本项目必须重点控制以下风险。

## 19.1 SQL 安全

禁止：

```text
DROP
DELETE
UPDATE
INSERT
ALTER
CREATE
COPY
ATTACH
INSTALL
LOAD
```

只允许：

```text
SELECT
WITH
```

------

## 19.2 文件安全

上传文件限制：

1. 只允许 `.csv`、`.xlsx`、`.xls`。
2. 限制文件大小，例如 20MB。
3. 文件名不直接作为保存路径。
4. 使用 uuid 重命名。
5. 文件保存在固定目录。
6. 禁止路径穿越。

------

## 19.3 工具安全

Agent 不应拥有无限能力。

第一阶段只开放：

```text
get_dataset_schema
query_data
profile_data
generate_chart
```

报告生成、文件写入、代码执行等能力应后续谨慎开放。

------

## 19.4 Python 执行安全

不要让模型直接执行任意 Python 代码。

错误示例：

```text
让模型生成 Python 代码，然后 exec 执行。
```

推荐做法：

```text
提供有限的分析工具，例如 correlation_analysis、missing_value_analysis、outlier_detection。
```

# 23. 项目最终形态

最终系统应成为一个完整的数据分析智能体平台。

用户不需要写 SQL，也不需要写 Python，只需要上传数据并用自然语言提问。

系统通过 LangGraph 控制 Agent 推理流程，通过 LangChain Tool 调用数据分析能力，通过后端服务执行安全可靠的数据处理，最终向用户返回：

1. 数据解释。
2. 查询结果。
3. 分析结论。
4. 可视化图表。
5. 报告文件。

本项目的重点不是“把旧项目迁移到 LangChain”，而是“从 0 开始构建一个真正基于 LangChain / LangGraph 的数据分析智能体项目”。
